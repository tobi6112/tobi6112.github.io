The `@Transactional`-Kotlin Mismatch

Recently, I’ve worked with a Spring Boot project written Kotlin, which had unexplainable strange behaviour by just looking at the code. Specifically, it is a game within a microservice architecture that is being developed as part of a course. One of the services is the so called “Game service” which is responsible for managing the state of the game and letting players (which are services by themselves) join a game. Once a player has joined a game, a asynchronous communication channel is created for the player as we tend to develop the service landscape in an event-driven architecture.
